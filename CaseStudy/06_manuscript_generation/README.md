# Manuscript Generation Records

This folder contains the input files and output records for the manuscript generation module of MacAma.

## Workflow Inputs

The manuscript generation workflow uses two primary input files:

- `user_manuscript_input.json`  
  User-provided manuscript information, including the research topic and section-specific details for Introduction, Materials and Methods, Results, and Discussion.

- `knowledge_base.json`  
  Local article-level knowledge base used for HyDE-enhanced BM25 retrieval. Each record contains article metadata such as title, authors, abstract, and DOI where available.

## Citation Retrieval

In the current case study, the retrieval parameter was set to `TOP_N_ARTICLES = 20`. The workflow first expands the research topic using HyDE, then retrieves the top 20 relevant articles from the local knowledge base using BM25. The retrieved articles are formatted as reference strings and supplied to the Writer Agent.

## Generation Workflow

The workflow generates the manuscript in the following order:

1. Introduction
2. Materials and Methods
3. Results
4. Discussion
5. Title
6. Abstract

For the Materials and Methods and Results sections, structured JSON inputs are first converted into narrative descriptions before section generation.

Each main section is generated through an iterative loop involving generation, expert-style review, decision judgment, and revision, with a maximum of three revision rounds.

## Output Files

- `ai_generated_manuscript.md`  
  Manuscript-style draft generated by the workflow using Qwen-Max.

- `human_revised_manuscript.md`  
  Version revised by human researchers after reviewing and editing the AI-generated draft.

## Notes

The AI-generated manuscript should be regarded as a draft rather than a publication-ready article. Human researchers remain responsible for verifying factual accuracy, checking citations, interpreting statistical results, revising arguments, and approving the final manuscript.
